Database Connection Info

This document contains the connection info for currently deployed databases in dr-eks-dev kubernetes cluster.

Note: You need to connect to DataReachable VPN first, because all databases are internal only.

  1. Redis

The in-cluster Redis is deployed in cluster mode. Use ioredis NodeJS driver to connect to the Redis cluster.

Currently, the deployed Redis has three master nodes, i.e. 3 shards, without any replicas. The hostnames and ports for the three master nodes:

  • redis-cluster-0.redis-cluster-headless.redis.svc.cluster.local:6379
  • redis-cluster-1.redis-cluster-headless.redis.svc.cluster.local:6379
  • redis-cluster-2.redis-cluster-headless.redis.svc.cluster.local:6379

The Redis service endpoint is: redis-cluster.redis.svc.cluster.local:6379. When connecting with redis-cli, make sure to attach the -c option.

Using ioredis to connect to this cluster:

const cluster = new Redis.Cluster([
  {
    port: 6397,
    host: "redis-cluster-0.redis-cluster-headless.redis.svc.cluster.local",
    password: "aCudZScHNN65ErvK"
  },
  {
    port: 6397,
    host: "redis-cluster-1.redis-cluster-headless.redis.svc.cluster.local",
    password: "aCudZScHNN65ErvK"
  },
  {
    port: 6397,
    host: "redis-cluster-2.redis-cluster-headless.redis.svc.cluster.local",
    password: "aCudZScHNN65ErvK"
  },
]);

The default root password to the Redis cluster is: aCudZScHNN65ErvK. You may create your own Redis ACL rules with username and password, but please don't change the root password before notifying others.

  1. MongoDB

The in-cluster MongoDB is deployed in sharded mode. Use appropriate MongoDB connection string to connect.

Currently, the deployed MongoDB has 2 mongos router instances, 2 data shard instances, 1 config server instance. The application should connect to the mongos instances to access the MongoDB cluster. In kubernetes, this is equivalent to connecting to the mongos service.

The connection string is: mongodb://mongodb-sharded.mongodb.svc.cluster.local:27017/. If you only want to do some experiments, please connect to the test database only so as not to ruin other in-use databases.

Note: The above string is the minimal connection string that only gives you access to the MongoDB cluster, but it does NOT mean you have appropriate configurations for your database/collections. For additional options and how to use a MongoDB sharded cluster, please refer to MongoDB Sharding.

For detailed user information, please check MongoDB User Setup.
