Kubernetes Statefulset PVC Expansion (Disk volume expansion)

Most database applications running on Kubernetes rely on the k8s StatefulSet resource. Changing volume size in such stateful applications is a routine procedure. However, resizing the PVC (persistent volume claim) resource is forbidden by the StatefulSet. Errors occur if one directly expands the volume size in the PVC resource (Shrinking the volume is NOT allowed).

Without tearing down the whole StatefulSet to avoid service downtime, follow these steps:

  1. kubectl get sts <name> -o yaml > old-sts.yaml for storing the current stateful set resource.
  2. kubectl edit pvc <name> for each PVC in the StatefulSet, to increase its capacity.
  3. kubectl delete sts --cascade=false <name> to delete the StatefulSet and leave its pods.
  4. kubectl apply -f old-sts.yaml to recreate the StatefulSet.

One must modify the PVC for each pod one-by-one.

If you want to monitor what's happening, run two more shell windows with these commands, before any of the commands above:

  • kubectl get pod -w
  • kubectl get pvc -w

There is no need to restart the pods, online pods can expand the file system without downtime. This works on k8s v1.18.

Note: for aws ebs volumes, aws ONLY allows one expansion within 6 hours.
