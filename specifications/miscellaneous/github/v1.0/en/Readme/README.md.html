Description

This repo defines the specifications on writing the README.md files at DataReachable. In each application source code repo and infrastructure code repo, this specification should be enforced.

List of Specifications

  • Title (Required)

    The title of the project should be placed at the first line and in Markdown Level 2 title:

    Optional: The contents of the title section should contain a centralized graph and relevant badges from Shields

  • Description (Required)

    The description of the project should be Markdown Level 2 title.

    The contents of the description should be Markdown normal text.

  • Project Structure (Required)

    The project structure should be Markdown Level 2 title.

    Subsections (Required)

    Subsection titles should be Markdonw Level 3 title. Possible subsections are listed below:

    1. Code Structure Overview (Required)
      The tree structure of the project's source code can be automatically generated by the VS Code extension: directory-tree generator.
    2. Root Folder Description (Optional)
      This section should include the detailed descriptions of each key file in the project root folder.
    3. Src Folder Description (Optional)
      This section should include the detailed descriptions of each key folder/file in the project src folder.
    4. Other Folder Description (Optional)
      Descriptions of other folders, such as testing, configuration etc.

  • Usage (Required)

    This section should include the usage of the project.

    Before describing the usage, instructions on how to configure a local environment should be included, such as must-install dependencies, minimum version requirement, etc.

    The contents of this section varies from project to project, but it should be well organized into a tree structure with example code given.

  • Change Log (Optional)

    This section should NOT be included in the README.md file, but should be recorded in another file CHANGELOG.md.

    Using the Auto ChangeLog Tool (Required)

    The change log is relied on the git logs, and should be automatically generated using the Auto ChangeLog tool.

    To use the Auto ChangeLog tool, first install it via npm install -g auto-changelog. Then, run the following command to generate the CHANGELOG.md file.

    auto-changelog --hide-credit --unreleased --output CHANGELOG.md

    This tool mainly logs the change logs of the formal releases. In each release, detailed changes should be included in the description of the release.

    The above command also logs all the unreleased commits.

    When using the above tool, the commit history is important. Therefore, each commit message should be formated for easy-reading. Refer to: Git Usage Conventions.
