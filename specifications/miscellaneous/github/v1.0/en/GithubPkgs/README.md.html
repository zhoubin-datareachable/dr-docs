GitHub Package Usage Specifications

At DataReachable, we store various packages/artifacts into github packages. Currently, github packages support Docker,NPM,Maven,NuGet and RubyGems. We mainly use docker and npm.

We prefer github packages to other package registries because github package provides private registry and we have paid github plans to support this. Using other private package registries incur additional fees. For public packages, it's ok to choose other registries, but we always keep the packages secret.

For docker images, we use GitHub Container Registry instead of the legacy GitHub Package Docker Registry (see migration for details), though ghcr.io is current beta version.

Naming Conventions

Each package should always be associated with a github repo.

The names of the packages should start with the names of the github repo.

As a github repo may be attached with multiple packages of the same type, an optional suffix is allowed to add to the package name.

The package names should all be in lowercase. (This might look weird and lengthy)

For repo naming conventions, see repo naming for details.

Examples:

Package Name Associated Repo Name
dr_front_questionnaireeditor dr_Front_QuestionnaireEditor
dr_back_usersystemsignup dr_Back_UserSystemSignup
dr_back_docserver dr_Back_DocServer
dr_back_docserver_nginx dr_Back_DocServer
