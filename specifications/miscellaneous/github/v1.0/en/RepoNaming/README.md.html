GitHub Repo Naming Conventions

  1. The repo name is in snake_case naming style
  2. The repo name should be prefixed with dr
  3. The repo name should ONLY have 4 underscores (prefix + 3 sections) at most, and 3 underscores (prefix + 2 sections) at least
  4. Each section is in PascalCase naming style
  5. The first section represents the repo category, must be one of the followings:
  • Front: A Frontend Project
  • Middle: A Middleend Project
  • Back: A Backend Project
  • DevOps: A DevOps Project
  • Doc: A Documentation Project
  • Side: A Side Project
  1. The second section is the project name in PascalCase. For sub-projects, extend the name, but do NOT add additional underscores
  2. The third section is an optional customised flag
  3. For prototypes, i.e. proof-of-concept projects, add a suffix Prototype as the third section to distinguish it from formal project repos

Examples

  1. dr_Front_UserSystemProfile
  2. dr_Back_UserSystemSignup
  3. dr_DevOps_UserSystemDashboard_Live
  4. dr_DevOps_QuestionnaireEditor_Modules
  5. dr_Side_LocalStorageFrontend_Prototype
