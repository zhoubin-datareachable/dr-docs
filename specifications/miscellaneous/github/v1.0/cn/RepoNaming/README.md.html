GitHub 仓库的命名规则

  1. 仓库名必须使用snake_case的风格
  2. 仓库名应该以dr作为前缀
  3. 仓库名应该仅仅包含最多四个下划线（前缀+3个部分），最少包含三个下划线（前缀+2个部分）
  4. 每个部分的名称必须使用PascalCase的风格
  5. 第一个部分是仓库分类，必须为下述几项中的一个：
  • Front: 前端项目
  • Back: 后端项目
  • DevOps: 中端项目
  • DevOps: 运维项目
  • Doc: 文档项目
  • Side: 非正式的其他项目
  1. 第二个部分是项目名称。对于子项目，扩展名称，不要添加多余的下划线
  2. 第三部分是一个可选的自定义标志
  3. 对于原型，也就是概念验证项目，使用后缀Prototype作为第三部分来将此项目与正式的开发项目区分开

例子

  1. dr_Front_UserSystemProfile
  2. dr_Back_UserSystemSignup
  3. dr_DevOps_UserSystemDashboard_live
  4. dr_DevOps_QuestionnaireEditor_modules
  5. dr_Side_LocalStorageFrontend_Prototype
