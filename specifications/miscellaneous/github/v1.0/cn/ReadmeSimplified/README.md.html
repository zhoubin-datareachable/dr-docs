简化版的README.md书写规范

规范列表

  • 项目标题

    ⚠️ 必须

    项目标题应放在第一行，使用Markdown第二级标题样式：

    可选项：项目标题下方可以放置一个居中的图片用以描述此项目，也可以包含相关的GitHub的徽章 Shields。

  • 项目描述

    😄 可选

    在标题下面，对项目加以简短描述。

    项目描述的内容使用Markdown的正常字体样式。

    项目描述并非强制要求，但是，最好提供。

  • 项目功能

    😄 可选

    一个功能清单或者功能表格。比如，项目中对外暴露的全部API。

    内容使用Markdown的正常字体样式。

  • 项目环境变量

    ⚠️ 必须

    环境变量的表格。在每个项目中，这是一个重要而且必须提供的部分，因为运维人员会根据项目的环境变量进行配置上线。

    每一个环境变量需要包括：变量名称，变量描述，和默认值。

    内容使用Markdown的正常字体样式。
