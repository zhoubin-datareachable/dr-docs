GitHub Package 使用规范

在数支公司，我们将各种产出的包、Docker镜像等存储在github包管理器中。目前，github包管理器支持Docker，NPM，Maven，NuGet 和 RubyGems。我们主要使用Docker和NPM。

我们使用github包管理器，而不是其他的registry，因为Github包管理器提供了私有仓库，而且公司已经使用付费版GitHub来支持相关功能。使用其他的私有包管理仓库会产生额外费用。对于公共包，使用其他的包管理仓库是可以的，但是我们总是隐藏我们的包。

对于Docker镜像，我们使用GitHub Container Registry代替GitHub Package Docker Registry(参见迁移中的细节)，虽然ghcr.io目前是beta版本。

命名习惯

每个包都应该对应一个GitHub仓库。

包的名字应该使用GitHub仓库的名字开头。

因为一个GitHub仓库可能关联多个同类型的包，我们允许在包名称的结尾多添加一个可选的后缀。

包名字应该都是小写的。（虽然，有的时候可能看起来十分奇怪和冗长）

对于仓库的命名习惯，参考仓库命名。

例子：

包名称 关联的仓库名称
dr_front_questionnaireeditor dr_Front_QuestionnaireEditor
dr_back_usersystemsignup dr_Back_UserSystemSignup
dr_back_docserver dr_Back_DocServer
dr_back_docserver_nginx dr_Back_DocServer
