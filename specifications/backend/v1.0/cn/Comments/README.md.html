源代码中的注释规范

源代码中的注释需要遵循在TS-Doc中定义的注释规范和样式，这样就可以通过TypeDoc这样的工具来从注释中生成HTML格式的文档。

注释样式

我们强制使用TypeScript的注释风格，如下：

/** 这是TypeScript样式的注释 */

避免使用C++风格的单行注释

// 这是C++风格的注释

需要被注释的语言成分

程序中的关键元素需要被加上注释：

  • 文件的元信息（题目，作者，创建日期，详细的描述）
  • 类、接口、类型定义中的属性和成员函数
  • 独立的函数和变量
  • 关键部分的逻辑
  • 特殊的设计考虑

备注：每一个源代码文件必须以文件的元信息作为开头，其中须包含题目，作者，创建日期，和详细的描述。

例子

/**
 * 样例文件
 * 作者：你的名字
 * 创建日期：01/01/1970
 * 描述：文件的详细描述
 */

 export class Statistics {
  /**
  * 返回两个数的平均值
  *
  * @remarks
  * 此方法是下述库的一部分：{@link core-library#Statistics | Statistics subsystem}.
  *
  * @param x - 第一个输入值
  * @param y - 第二个输入值
  * @returns `x`和`y`的算术平均值
  *
  * @beta
  */
  public static getAverage(x: number, y: number): number {
    return (x + y) / 2.0;
  }
}
