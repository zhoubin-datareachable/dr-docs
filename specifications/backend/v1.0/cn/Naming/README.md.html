命名习惯

命名习惯对于让代码仓库保持一致和易于维护是十分重要的。我们规定了一组命名习惯，这些命名习惯由ESLint检查。包括：

  • 文件名风格
  • TypeScript源码中的命名风格

文件夹和文件的命名风格

文件夹和文件必须使用**snake_case**风格命名，比如ctr_sample_controller.ts, rte_sample_router.ts, sample_view.ejs, sample_style.css。不能使用其他风格。

ESLint检查文件夹和文件的命名风格。错误的命名风格会被ESLint报告为错误。

源代码中的命名风格

TypeScript语言中需要命名的元素可以被分类如下：

  1. variable - 匹配任何var / let / const修饰的变量名。
  • 允许的修饰符: const, destructured, global, exported, unused.
  • 允许的类型: boolean, string, number, function, array.
  1. function - 匹配任何命名函数生命和命名函数表达式。
  • 允许的修饰符: global, exported, unused.
  • 允许的类型: none.
  1. parameter - 匹配任何函数参数。并不匹配参数的属性。
  • 允许的修饰符: destructured, unused.
  • 允许的类型: boolean, string, number, function, array.
  1. classProperty - 匹配任何类中的属性。并不匹配赋值了函数表达式的类属性。
  • 允许的修饰符: abstract, private, protected, public, readonly, requiresQuotes, static.
  • 允许的类型: boolean, string, number, function, array.
  1. objectLiteralProperty - 匹配任何对象字面量中的属性。并不匹配赋值了函数表达式的对象属性。
  • 允许的修饰符: public, requiresQuotes.
  • 允许的类型: boolean, string, number, function, array.
  1. typeProperty - 匹配任何使用对象定义的类型中的属性。并不匹配赋值了函数表达式的对象属性。
  • 允许的修饰符: public, readonly, requiresQuotes.
  • 允许的类型: boolean, string, number, function, array.
  1. parameterProperty - 匹配任何参数属性
  • 允许的修饰符: private, protected, public, readonly.
  • 允许的类型: boolean, string, number, function, array.
  1. classMethod - 匹配类中的方法。同时，也匹配类中赋值了函数表达式的类属性。并不匹配访问器函数。
  • 允许的修饰符: abstract, private, protected, public, requiresQuotes, static.
  • 允许的类型: none.
  1. objectLiteralMethod - 匹配任何对象字面量中的方法。同时匹配赋值了函数表达式的属性。并不匹配访问器函数。
  • 允许的修饰符: public, requiresQuotes.
  • 允许的类型: none.
  1. typeMethod - 匹配任何对象定义类型中的属性。同时匹配赋值了函数表达式的属性。并不匹配访问器函数。
  • 允许的修饰符: public, requiresQuotes.
  • 允许的类型: none.
  1. accessor - 匹配任何访问器函数。
  • 允许的修饰符: abstract, private, protected, public, requiresQuotes, static.
  • 允许的类型: boolean, string, number, function, array.
  1. enumMember - 匹配任何枚举类型中的成员。
  • 允许的修饰符: requiresQuotes.
  • 允许的类型: none.
  1. class - 匹配任何类声明名称。
  • 允许的修饰符: abstract, exported, unused.
  • 允许的类型: none.
  1. interface - 匹配任何接口声明名称。
  • 允许的修饰符: exported, unused.
  • 允许的类型: none.
  1. typeAlias - 匹配任何类型别名声明。
  • 允许的修饰符: exported, unused.
  • 允许的类型: none.
  1. enum - 匹配任何枚举类型声明。
  • 允许的修饰符: exported, unused.
  • 允许的类型: none.
  1. typeParameter - 匹配任何泛型类型声明中的类型参数。
  • 允许的修饰符: unused.
  • 允许的类型: none.

源代码中所有允许的命名格式的总结：

  • strictCamelCase - 标准的 camelCase 格式 - 字母之间不允许有下划线，同时连续的大写字母也是不允许的，比如：myId是可以的，但是myID不可以。
  • StrictPascalCase - 与strictCamelCase相同，除了第一个字母必须是大写的之外。
  • snake_case - 标准的snake_case格式 - 所有的字母必须是小写，而且可以有下划线。
  • UPPER_CASE - 与snake_case格式相同，除了所有的字母必须都是大写的之外。

在源码中，我们针对不同类别的代码元素，使用如下的具体命名风格：

格式 是否允许以下划线开头 是否允许以下划线结尾 对应的元素
strictCamelCase ❌ ❌ function, classMethod, typeMethod, accessor
strictCamelCase ❌ ❌ variable with "function" type, e.g. arrow functions assigned to a variable
UPPER_CASE, snake_case ✅ ✅ variable with "const" modifier
snake_case ✅ ❌ variable, parameter, classProperty, parameterProperty, typeProperty
StrictPascalCase ❌ ❌ class, interface, typeAlias, enum, typeParameter
UPPER_CASE, StrictPascalCase ❌ ❌ enumMember

objectLiteralProperty 和 objectLiteralMethod 被以上规则排除在外，为了与已有的JavaScript库兼容。
