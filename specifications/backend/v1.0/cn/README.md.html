数支公司后端开发规范

本文档规定了在数支公司进行后端开发时应该遵循的规范。本文档的目的是为了防止开发人员随意采用项目文件布局、命名格式和配置等。此文档是一份全面的后端开发规范，包括目前在数支公司使用的所有技术栈。

开发环境要求

重要： 对于后端开发，我们强制开发中使用的操作系统为Ubuntu LTS 版本 (目前最新为20.04)，或者MacOS。禁止使用Windows。

对于开发中使用的IDE，你可以从VSCode（严格来讲不是IDE）和JetBrain（比如，WebStorm，IntelliJ等）中选择。我们推荐使用VSCode，以保持跟组内其他成员一致。不过，使用JetBrains也可以，只要你最终符合开发规范中的规定即可。

当前使用的技术栈

在当前第一阶段的开发中，主要使用的开发技术栈是：NodeJS以及其上的ExpressJs框架。所使用的开发语言为TypeScript。未来，此技术栈可能会发生更改，也可能引入其他新的后端开发技术栈。

  • NodeJS： 使用当前最新的LTS版本（v14.x）以保证稳定性和最新的NodeJS特性
  • ExpressJs： 使用当前的稳定版本（v4.x）
  • TypeScript：使用当前稳定的版本（v4.x）
  • Redis：使用版本 v6.0.10
  • MongoDB：使用版本 v4.4.x
