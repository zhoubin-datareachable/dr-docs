后端开发代码审查流程

对于后端开发项目，通常，我们使用三个不同类型的分支，主分支，功能分支，和开发分支。开发人员应当仅使用开发分支工作。主分支由项目经理管理，功能分支由团队主管管理。

另外，快速修补分支 由团队主管负责管理，开发人员可以直接在该分支上工作，并向main分支直接提交Pull Request。

分支类型 分支管理员 谁向该类分支提交Pull Requests? 是否需要代码检查? 谁来做代码检查？
主分支 项目经理 团队组长 ✅ 项目经理
功能分支 团队组长 开发人员 ✅ 或者 ❌ (团队组长安排) 团队组长
快速修补分支 团队组长 开发人员 ✅ 项目经理 或者 团队组长

团队组长和项目经理也可以作为开发人员贡献代码。当他们作为开发人员的角色时，他们也应该遵守上面的限制。
