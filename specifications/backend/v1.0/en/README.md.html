DataReachable Backend Development Specifications

This documentation sets out the best practices that should be followed when developing backend applications at DataReachable. The purpose of this documentation is preventing ad hoc project folder layouts, naming styles, configurations, etc. This is a comprehensive backend development specification including all tech stacks that are currently used at DataReachable.

Development Environment Requirements

IMPORTANT: For backend development, we enforce the operating system used in development to be Ubuntu LTS version (currently 20.04), or MacOS. Windows is forbidden.

For developing IDE, you can choose from VSCode (not an IDE strictly) and JetBrains (e.g. Webstorm, IntelliJ etc.). We recommend using VSCode to be consistent with the team members, but using JetBrains is also alright as long as you comply with the specifications.

Current Tech Stack(s)

For the first development stage, the primary tech stack used at DataReachable is NodeJS with the ExpressJs as the development framework and TypeScript as the programming language. In the future, this tech stack is subject to change and other tech stacks might be included.

  • NodeJS: Use the current LTS version (v14.x) for both new features and stability.
  • ExpressJs: Use the current stable version (v4.x).
  • TypeScript: Use the current stabel version (v4.x)
  • Redis: Use version v6.0.10.
  • MongoDB: Use version v4.4.x.
