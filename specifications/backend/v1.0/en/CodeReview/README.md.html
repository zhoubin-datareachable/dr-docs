Backend Development Code Review Process

For a backend dev project, normally, we have three different types of branches, the main branch, the feature branch and the dev branch. Developers should work on their own dev branch. The main branch is managed by the project managers, and the feature branch is managed by team leaders.

In addition, the hotfix branch is managed by team leaders, developers can directly work on the hotfix branch, and submit pull requests to the main branch.

Branch Type Branch Manager Who Submits Pull Requests? Required Code Reviews? Code Reviewers
Main Project managers Team leaders ✅ Project managers
Feature/** Team leaders Developers ✅ or ❌ (team leaders decide) Team leaders
Hotfix/** Team leaders Developers ✅ Project managers or team leaders

Team leaders and project managers can also be developers to contribute code. When they are developers, they should also follow the above restrictions.
