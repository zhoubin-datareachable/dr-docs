GitOps Best Practices

At Datareachable, we adopt GitOps to automate the DevOps process and bridge the gap between development and deployment.

Concept Explanation

Simply speaking, the idea of GitOps is storing both application code and infrastructure code in GitHub repos and considering these repos as the single source of truth. Via certain DevOps tool, both repos are seamlessly glued together and the process from development to deployment is fully automated. For details, refer to GitOps.

WorkFlow Overview

We use GitHub Actions as our choice of the above DevOps tool and AWS Elastic Kubernetes Service as the cloud service, and the overall workflow is as follows:


Overview of GitOps WorkFlow

Application developers commit and push source code into applciation repo with a Dockefile. On releasing new versions, a github action is triggered to build a docker image and push into the github package registry. Once done, the DevOps developers are responsible for deploying the built docker images into the online kubernetes cluster.
