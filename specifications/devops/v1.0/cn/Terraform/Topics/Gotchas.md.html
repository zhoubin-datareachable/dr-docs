其他杂项

1. 版本锁定

你应该锁定某个terraform模块中使用的Terraform工具的版本，因为terraform工具自身还在快速开发迭代中。Terraform每次新版本发布并不考虑向后兼容，比如：从terraform 0.11.x版本升级到0.12.x版本需要更改很多使用0.11.x版本编写的代码。锁定版本可以避免偶然犯错误。

使用如下代码段锁定需要的terraform版本：

terraform {
  # Require any 0.13.x version of Terraform
  required_version = ">= 0.13, < 0.14"
}

你也应该锁定所有的terraform中使用的provider的版本，比如，对于适用于AWS的provider：

provider "aws" {
  region = "ap-southeast-2"
  # Allow any 3.x version of the AWS provider
  version = "~> 3.0"
}

2. 在部署前调用输出更改计划命令

应该总是先运行 terraform plan (或者 terragrunt plan) 命令，然后，仔细检查输出，确认更改计划是符合预期的。之后，才能运行 terraform apply (或者 terragrunt apply) 命令，应用更改计划。
