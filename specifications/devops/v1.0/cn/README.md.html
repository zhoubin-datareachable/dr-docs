DataReachable DevOps 规范

此GitHub仓库列出了在DataReachable进行DevOps时应遵循的最佳实践。这个规范的目的是防止开发人员随意选择工具链，制定项目文件夹布局，配置等。设置规则和约定时，应充分考虑到未来的要求和灵活性，而不是为了满足一时的需要。随着DevOps行业的深入发展，此GitHub仓库会逐步完善和迭代，但是，更改和重构需要以增量的形式逐步推进，不建议做出重大变更。

注意： DevOps技术本身仍处于起步阶段，工具链等也存在重大变更的可能性，因此，在目前，此规范仍然仅仅是初步尝试。

主要的技术栈

  • 核心架构: 微服务
  • 容器化工具: Docker
  • 辅助工具: 自动化集成和部署工具 (Github Action)

主要开发使用的工具链

  • 云计算服务使用: AWS
  • 容器编配工具: Kubernetes
  • 基础设施即代码（IaC）工具: Terraform 和 Terragrunt

虽然主要的技术栈采用基于Docker的容器化方案，但是我们并不会不考虑其他的方案以保证DevOps的灵活性和满足未来发展需求。

Terragrunt 是Terraform工具的一个扩展工具，用来弥补Terraform在搭建项目中的一些不足（Terragrunt底层仍然是调用Terraform）。

基本工作流

对于代码的开发工作流定义如下：

  • 应用程序开发工作流
  • 基础设施代码开发工作流

重要:

一旦开始使用Terraform和Terragrunt工具链，应该仅仅使用这两种工具来操控实际部署的基础设施。其他的操控方式应该被禁止以防止可能出现的不一致性，比如，手动在AWS控制台内（AWS的可视化资源操作网页）更改基础设施。
