前端常见 Q&A

该文档整理了在前端开发过程中常见的问题，以及问题的解答

Q: 每次拿到新的任务开发完成后，代码 push 到了 github，这时候再 checkout 一个新的分支完成另外一个任务，随后再发 pull request 就会出现 conflict

A: 每次拿到开发任务以后需要重新从 development checkout 新的分支出来，随后在这个新的分支上进行任务开发，这时候如果你的任务完成以后，如果直接在当前分支又 checkout 一个分支出来继续开发，这时候你的 pull request 一定会有 conflict，重点在于每次拿到任务一定要从 development 上面 checkout 新的分支，一定要从 development 上面 checkout 新的分支，一定要从 development 上面 checkout 新的分支，重要的事情说三遍。

Q: 在不同设计尺寸下版面的控制是写死尺寸，还是用百分比来表示尺寸？

A: 目前设计的尺寸分为四个断点，375px, 768px, 1024px 和 1920px

  1. 375px：当屏幕宽度小于 768px 的时候，按照 375px 的设计，并且框架的宽度以 100%计算。
  2. 768px：当屏幕宽度在 768px 和 1024px 之间的时候，屏幕设计按照 768px 的设计，并且框架的宽度以 100%计算。
  3. 1024px：当屏幕的宽度在 1024px 和 1600px 之间的时候，屏幕设计按照 1024px 的设计，并且框架的宽度以设计图宽度为准，不以百分比进行计算。
  4. 1920px：当屏幕的宽度大于 1600px 的时候，屏幕设计按照 1920px 设计进行，并且在屏幕宽度超过 1920px 的时候对屏幕进行等比缩放。

注：如果设计中没有 1920px 的设计样式，则在屏幕尺寸大于 1024px 以后就对样式进行等比缩放，在屏幕尺寸大于 1024px 以后全部以设计图的尺寸为准，宽度不以百分比书写。如果设计中不存在 ipad 于手机端设计，则只考虑屏幕尺寸大于等于 1024px 的情况，其余情况当前开发阶段不予以考虑。

Q: 关于 css 全局样式的配置是什么？

A: 目前项目中基本所有的样式文件都会设置这样一个样式 _ {margin:0; padding:0} 为了保证代码的整洁性和可维护性，目前所有的项目都会在 src 目录下添加 global.scss 的全局样式，并且把_{margin:0;padding:0}的样式配置到 global.scss 文件中。其余的一些通用样式请大家通过 class,id 以及其他 css 选择器的方式对不同样式进行单独调整。

Q: 在从 github 拉取代码以后，想要重命名之前的文件夹首字母为大写，但是 github 却提示没有任何修改，也无法将新命名的大写文件 commit 到远程分支，这时候怎么办？

A: 首先你需要在 terminal 中运行 git config --get core.ignorecase 这段指令，看看是否输出为 true，如果为 true 则运行这段指令来修改 github 大小写敏感 git config core.ignorecase false, 这时候再次运行 git config --get core.ignorecase，如果返回为 false 则代表修改成功，github 就可以进行大小写敏感的处理了。不过需要注意的是，这时候 push 到远程服务器可能会出现两个相同的文件夹，即大写命名文件夹一个和小写命名文件夹一个，请联系部门主管来修改远程服务器中的文件夹结构。
