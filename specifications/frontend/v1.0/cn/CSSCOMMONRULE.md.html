Q: 每次拿到新的任务开发完成后，代码 push 到了 github，这时候再 checkout 一个新的分支完成另外一个任务，随后再发 pull request 就会出现 conflict

A: 每次拿到开发任务以后需要重新从 development checkout 新的分支出来，随后在这个新的分支上进行任务开发，这时候如果你的任务完成以后，如果直接在当前分支又 checkout 一个分支出来继续开发，这时候你的 pull request 一定会有 conflict，重点在于每次拿到任务一定要从 development 上面 checkout 新的分支，一定要从 development 上面 checkout 新的分支，一定要从 development 上面 checkout 新的分支，重要的事情说三遍。
