Specifications for Front End Project

This readme file is the specifications for the front end. Please see the following list of all the specifictions for front end part.

List of Markdown specifications

  • Title

    The title of the project should be Markdown Level 1 title:

    Optional: The contents of the title section should contain a centralized graph and relevant badges from Shields

  • Description

    The description of the project should be Markdown Level 1 title.

    The contents of the description should be Markdown normal text.

  • Project Structure

    The project structure should be Markdown Markdown normal text.

    Subsections:

    Subsection titles should be Markdonw Level 2 title.

    1. Code Structure Overview
      The tree structure of the project's source code can be automatically generated by the VS Code extension: directory-tree generator.
    2. Root Folder Description
      This section should include the detailed descriptions of each key file in the project root folder.
    3. Src Folder Description
      This section should include the detailed descriptions of each key folder/file in the project src folder.
    4. Other Folder Description
      Descriptions of other folders, such as testing, configuration etc.

Usage

  • setup your project in your local environment

    The SAMPLEPROJECT.md file is mianly introduce the front end project structure and how to set up this project environment in your local computer.

  • general coding rules

    The PROJECTRULES.md file is mainly describe the general rules during coding. Such as how to name the class and how to write the comment in your project.

  • git usage

    The GITSPECIFICATIONS.md file is mianly describe the rule of how to use git, such as how to name a new branch and how to write a commit message.

  • code review standard

    The CODEREVIEW.md mainly include the code review rules, it's about how to do the code review between different team members.

  • testing standard

    The TESTINGRULE.md mainly include the testing rules.

  • redux standard

    The REDUXRULES.md mainly include the redux coding rules.

  • saga standard

    The SAGARULES.md mainly include the saga coding rules.

  • Ajax request standard

    The AJAXREQUESTRULE.md mainly include the Ajax request rule in the project.

  • Front End Frequent Q&A

    Q&A.md mainly include the frequent Q&A during front end development.
