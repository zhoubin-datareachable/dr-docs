Front End Frequent Q&A

This document sorts out common questions in the front-end development process, and answers to questions

Q: Every time a new task is obtained and the development is completed, the code is pushed to github. At this time, a new branch is checked out to complete another task, and then a pull request is sent again. A conflict will appear.

A: Every time you get a development task, you need to checkout a new branch from the development branch, and then perform task development on this new branch. At this time, if your task is completed, if you directly checkout another branch in the current branch to continue development, At this time, your pull request must have conflict. The point is that every time you get a task, you must checkout new branches from development.

Q: What default attributes in css need to be added

A: 1. Any UI interaction styles appearing on the UI represent the interaction process between the user's mouse and the UI. At this time, you need to set the properties of the cursor according to the current interaction scene. 2. When some text exists in a button or a card that can be clicked, you need to set user-select to none for these texts.

Q: Under different design sizes, is the layout control hard-coded, or is the size expressed as a percentage?

A: The current design size is divided into four breakpoints, 375px, 768px, 1024px and 1920px

  1. 375px: When the screen width is less than 768px, follow the design of 375px, and the width of the frame is calculated as percentage.
  2. 768px: When the screen width is between 768px and 1024px, the screen design follows the 768px design, and the frame width is calculated as percentage.
  3. 1024px: When the width of the screen is between 1024px and 1600px, the screen design follows the design of 1024px, and the width of the frame is subject to the width of the design drawing, not calculated as a percentage.
  4. 1920px: When the width of the screen is greater than 1600px, the screen design will follow the 1920px design, and the screen will be scaled proportionally when the screen width exceeds 1920px.

Note: If there is no 1920px design style in the design, the style will be scaled proportionally after the screen size is greater than 1024px. After the screen size is greater than 1024px, all are subject to the size of the design drawing, and the width is not written as a percentage. If there is no ipad in the mobile design in the design, only the case where the screen size is greater than or equal to 1024px is considered, and the other cases are not considered in the current development stage.

Q: What is the configuration of css global style?

A: Currently, all style files in the project will set such a style * {margin:0; padding:0}, In order to ensure the cleanliness and maintainability of the code, all projects will add the global.scss file under the src directory, And configure the style of *{margin:0;padding:0} into the global.scss file. For the rest of the general styles, please adjust the different styles individually through class, id and other CSS selectors.

Q: After pulling the code from github, I want to rename the previous folder with capital letters, but github prompts that there is no modification and cannot commit the newly-named capital file to the remote branch. How to fix this issue?

A: First, you need to run the git config --get core.ignorecase command in the terminal to see if the output is true. If it is true, run this command to modify the github case-sensitive git config core.ignorecase false, run git config --get core.ignorecase again at this time, if it returns false, it means the modification is successful, and github can do case-sensitive processing. However, it should be noted that at this time, when pushing to the remote server, there may be two identical folders, one uppercase named folder and one lowercase named folder. Please contact the department head to modify the folder structure in the remote server.
