WireGuard VPN (/* TODO Translation */)

WireGuard VPN allows us to access the internal network through a bastion host.

Client Installation Instructions

  1. Install homebrew.
  2. Install WireGuard brew install wireguard-tools
  3. Obtain a keypair from Matthew or Travis
  4. Run the following WireGuard commands:
# To connect
wg-quick up <path to wireguard config file>
# To disconnect
wg-quick down <path to wireguard config file>
  1. If you have any questions, contact Matthew or Google.

Hints

Try adding ./ to the relative path of the file if wg-quick produces an error.

Users Notes

WireGuard only routes traffic to the following hosts through the VPN:

  • 10.6.0.0/24 (VPN internal network)
  • 10.100.0.0/16 (Kubernetes internal network)
  • 192.168.0.0/16 (VPC of dev/apse2)

As such, your other traffic (eg. stackoverflow.com) will still originate from your own IP address.

This may cause IP conflicts/reroutes with local resources at the Glenfield office.

Maintainers Notes

  • Generate new config files by running pivpn -a on the bastion host.
  • The generated WireGuard config file proxies the entire internet by default.
    • Please only select the specified IP range above before handing off the WireGuard config file.
  • In-cluster resource requests are funnelled through 10.6.0.3, which is a container under the kubernetes namespace wireguard-vpn
    • The container in the cluster is a client of the VPN server hosted on the bastion host.
    • The container must be ran on a host with wireguard installed. This is signified by the label tools.datareachable.net/vpn=wireguard
